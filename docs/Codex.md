# Codex – Leitfaden und Regeln

Dieses Dokument beschreibt Richtlinien und Verhaltensregeln für die Entwicklung des Taufspruch Finder.

## Allgemeine Prinzipien
- **Barrierefreiheit**: Die App soll für alle Nutzer zugänglich sein (Responsive Design, Screenreader‑Unterstützung).
- **Klarheit und Minimalismus**: Benutzeroberfläche sauber, ohne Ablenkungen. Wenige Schritte führen zum Ergebnis.
- **Theologische Sorgfalt**: Bibelverse und Kategorien sollen pastoral sensibel ausgewählt werden.
- **Offene Zusammenarbeit**: Beiträge willkommen. Bitte respektiere den Code of Conduct.

## Code‑Qualität
- Nutze moderne React‑Techniken (Hooks, Functional Components).
- Dokumentiere komplexe Funktionen.
- Schreibe Tests für kritische Komponenten.

## Beiträge
- Erstelle einen Fork, committe in feature branches und erstelle Pull Requests.
- Jeder PR sollte eine klare Beschreibung und Referenzen zu Tickets/Issues enthalten.
- Beachte Style‑Guidelines (Prettier, ESLint).

## Lizenzen
- Verwende keine urheberrechtlich geschützten Assets ohne Genehmigung.
- Stelle sicher, dass eingebettete Bibelübersetzungen frei verwendet werden dürfen oder verweise auf offizielle APIs.

## KI‑Integration
- KI‑Funktionen (semantische Suche, automatische Empfehlungen) sollten transparent und erklärbar sein.
- Die GenAI‑Modelle müssen den technischen Vorgaben (CUDA 12.8, PyTorch 2.9 etc.) entsprechen.
